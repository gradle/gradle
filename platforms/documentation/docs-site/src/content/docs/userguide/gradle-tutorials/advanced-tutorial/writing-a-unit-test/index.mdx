## Step 1: About Unit Tests

Gradle `init` creates a sample unit test to get you started.

It&#8217;s located in `src/test/kotlin/org/example/PluginTutorialPluginTest.kt`.

It&#8217;s located in `src/test/groovy/org/example/PluginTutorialPluginTest.groovy`.

The original test is simple, it applies the default `org.example.greeting` plugin and verifies that the `greeting` task is available.
We will update it to test our new plugin.

## Step 2: Update the Test

First, let&#8217;s rename the test file to match our plugin&#8217;s name.

Rename the file `PluginTutorialPluginTest.kt` to `SlackPluginTest.kt`.

Rename the file `PluginTutorialPluginTest.groovy` to `SlackPluginTest.groovy`.

Next, update the code to reflect our plugin&#8217;s ID (`org.example.slack`) and task name (`sendTestSlackMessage`).

{/* converter-gap: node=sample dir="snippets/tutorials/developing-plugins/plugin-tutorial/kotlin" files="plugin/src/test/kotlin/org/example/SlackPluginTest.kt[]" */}

{/* converter-gap: node=sample dir="snippets/tutorials/developing-plugins/plugin-tutorial/groovy" files="plugin/src/test/groovy/org/example/SlackPluginTest.groovy[]" */}

This test:

1. ***`ProjectBuilder.builder().build()`***: Creates an in-memory test project, which is perfect for isolating our plugin&#8217;s behavior.
2. ***`project.plugins.apply(&#8230;&#8203;)`***: Applies our `SlackPlugin` to the test project.
3. ***`assertNotNull(&#8230;&#8203;)`***: Confirms that our `sendTestSlackMessage` task was successfully registered by the plugin.

## Step 3: Run the Test

You can now run the `test` task from the command line to make sure everything is working correctly.

```bash
$ ./gradlew test
```
