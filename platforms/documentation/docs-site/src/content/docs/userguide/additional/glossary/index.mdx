There are three _kinds_ of plugin, based on the context in which they are applied:

1. Project plugins that implement `Plugin&lt;Project&gt;`, applied in [build scripts](#sub:terminology_build_script).
2. Settings plugins that implement `Plugin&lt;Settings&gt;`, applied in [settings scripts](#sub:terminology_settings_script).
3. Init (Gradle) plugins that implement `Plugin&lt;Gradle&gt;`, applied in [init scripts](#sub:terminology_init_script).

Plugins may be _implemented_ as so-called binary plugins (that is, by explicitly implementing one of the specific generic interfaces described above), or as [precompiled script plugins](#sub:terminology_precompiled_script_plugin).
This distinction is merely an implementation detail.
